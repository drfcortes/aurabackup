AURA JSON Block

The AURA JSON block is the core structure that allows any document or artifact to transparently declare the use of generative AI. This metadata ensures traceability, reproducibility, and ethical accountability. 

{
  "aura_version": "0.1",
  "model": "GPT-4o",
  "provider": "OpenAI",
  "prompt": "Summarize the paper",
  "timestamp": "2025-07-25T10:32Z",
  "user_edit": true,
  "content_hash": "a0f4...dc82",
  "license": "CC-BY-4.0",
  "notes": "Content edited",
  "uid": "AURA-2025-000001"
}

Field Definitions

aura_version
Version of the AURA protocol used for metadata generation.
model
Name of the AI model (e.g., GPT-4o, Stable Diffusion).
provider
The platform or service through which the AI was accessed (e.g., OpenAI, HuggingFace).
prompt
The human-written input that triggered the AI generation.
timestamp
Time of generation in UTC (ISO 8601 format).
user_edit
Boolean flag indicating whether the AI output was edited by a human after generation.
content_hash
Hash value representing the final content. Used to verify integrity and non-tampering.
license
Usage license of the content (e.g., CC-BY-4.0).
notes
Optional notes by the human contributor (e.g., type of edit, intent).
uid
Unique identifier for the contribution. Can be linked to a public database.

These fields can be extended to include additional provenance information depending on context or institution-specific requirements.
